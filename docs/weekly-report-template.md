# 週報テンプレート

週報作成時はこの形式に従う。過去の週報は `docs/weekly-reports/` に日付ファイル名（`YYYY-MM-DD.md`＝週の最終日）で保存する。

## 作成手順

1. `git log --since="<週の開始日>" --pretty="%ad %s" --date=format:"%m/%d"` で今週のコミットを確認する
2. 「今週の進捗」はコミット履歴を正とし、機能単位でまとめる（コミット1件＝1行ではない）
3. 「課題 / 相談事」には設計判断で迷った点・保留にした点を、判断の根拠つきで書く
4. 「学び」は実装の詳細ではなく、他プロジェクトでも使える一般化した気づきを書く

## 形式

```markdown
# YYYY/MM/DD 週報 — 小林相太

- 期間: YYYY/MM/DD 〜 YYYY/MM/DD
- チーム / プロジェクト: GochiSpace（個人開発）
- 関連チャンネル: #4th_vibe_coding

## Project 概要
- **GochiSpace**：家族や友人、知人など互いに知り合い同士の関係のみでユーザーがレストランのレビューを投稿・共有できるサービス。
- **なぜ**：既存の主要な飲食店レビューサイトは匿名性が高く、サクラレビューや個人の主観によるノイズが混在しており、情報の信頼性に欠けるという課題がある。身近で信頼できる人物の間で安心して飲食店を選べる仕組みが必要だと感じたため。
- **どこに向かって**：相互フォローによるクローズドな関係性のなかで完全に信頼できるレビューのみを共有し、味の好みや店舗の雰囲気を把握した上で、不安なく最適な飲食店選定ができる新たな体験の提供を目指す。

## 今週の進捗
**✅ 完了**
- [x] （完了した項目）
- [ ] （着手したが未完了の項目）

## 課題 / 相談事
- （設計判断で迷った点・保留にした点・フィードバックがほしい点）

## 次週の TODO
- [ ] （次週にやる予定の項目）

## 学び / シェアしたい情報
- 📝 （今週の気づき）
```

※「Project 概要」は毎週同じ文面を使い回す（サービスの方向性が変わったときのみ更新）
